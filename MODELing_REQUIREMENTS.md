# 🤖 3D大模型伴侣：表情与动作设计需求清单（极简版）
*(3D LLM Companion: Simplified Expressions & Animations Spec)*

为了实现最核心的声音同步（口型）、情感气场（表情）以及交互呼应（动作），请美术设计师重点保障并交付以下**必备清单**。

---

## 🗣️ 一、 核心表情与口型清单 (Essential Blendshapes)

模型头部必须包含以下 **15个最关键的混合变形（Blendshape）**，用于实时拼音口型驱动与基本神态微表情：

### 1. 自动眨眼与眼神 (4个)
* **`eyeBlinkLeft` / `eyeBlinkRight`** (左右独立眨眼):
  * **作用**: 在 Web 端自动运行随机眨眼算法，消除“机器人睁眼死板感”。
* **`eyeLookInLeft/Right` / `eyeLookOutLeft/Right`** (眼珠转动):
  * **作用**: 允许伴侣在特定交互时转动视线，跟随用户的鼠标。

### 2. 核心拼读口型 (6个)
用于将大模型输出的文本或语音波形实时转换成嘴型起伏（嘴唇必须有内部牙齿和简易口腔，防止黑洞穿模）：
* **`jawOpen`**: 下巴自然张开，是一切口型变幻的物理轴动力。
* **`mouthOpen`**: 嘴唇上下张开（发 `/a`、`/e`、`/o` 等大开口元音）。
* **`mouthPucker`**: 嘴唇朝中心用力聚拢嘟起（发 `/u` 口型，或闭嘴思考）。
* **`mouthFunnel`**: 嘴唇微圆向前突出（发 `/o`、`/w` 等喇叭口型）。
* **`mouthSmileLeft` / `mouthSmileRight`**: 嘴角微咧（发 `/i` 等扁平口型）。

### 3. 微表情与情绪反馈 (5个)
可以通过不同比例混合，拼好“喜、怒、哀、乐、无奈”：
* **`mouthSmileLeft` / `mouthSmileRight`** (全拉满): 日常开心大笑、温柔浅笑。
* **`mouthFrownLeft` / `mouthFrownRight`**: 嘴角耷拉。代表失落、难过、道歉或委屈。
* **`browInnerUp`**: 眉头向上耸起（呈“八”字眉）。代表委屈、伤感、令人怜悯。
* **`browDownLeft` / `browDownRight`**: 压低眉头、降眉。用于愤怒、严肃或极度专注。
* **`cheekPuff`**: 鼓腮帮子。伴侣代表性俏皮、撒娇或假装生气的可爱神态。

## 🏃 二、 核心动作动画剪辑 (Required Action Clips)

动作使用 **30 FPS 采样导出**，角色根骨骼（`Hips`）在空间上原地不动（In-Place），位移用代码控制，避免闪烁。

| 序号 | 动画 Clip 命名 | 循环属性 | 帧数/推荐时长 | 触发时机与肢体神态描述 |
| :-: | :--- | :-: | :-: | :--- |
| **1** | `idle_default` | **Loop** | ~180帧 (6s) | **待机呼吸**: 全身极微弱呼吸起伏，肩膀微耸，重心不规律在双脚极细微移动。双手自然垂于小腹前微握。 |
| **2** | `listening` | **Loop** | ~120帧 (4s) | **倾听状态**: 当用户在打字输入时触发。身体重心极轻微前倾，头部微微侧偏、双眼直视镜头并浅浅微笑。 |
| **3** | `thinking` | **Loop** | ~150帧 (5s) | **思考中**: 大模型正在生成回答时循环。单手微抬抚下颌，眼神微移看向斜上方，手指不规律轻点，表现心不在焉的专注深思。 |
| **4** | `talk_neutral`| **Loop** | ~240帧 (8s) | **基础陈述说话**: 说话时的主循环。双臂自然抬在身前起伏做摊手、五指舒展等叙述性手势，**下半身和嘴巴动作除外（口型由表情实时驱动）**。 |
| **5** | `talk_excited`| **Loop** | ~150帧 (5s) | **欢快激昂陈述**: 大模型表现喜悦或惊叹时。肢体张开幅度稍大、配合轻快点头或双手在胸前有短促开合动作。 |
| **6** | `greet` | **Once** | ~90帧 (3s) | **开场打招呼**: 初次进入页面或苏醒时。热烈向前跨迎半步、扬手快速挥舞打招呼，两秒后平滑插值淡入 `idle_default` |
| **7** | `sadness` | **Loop** | ~180帧 (6s) | **伤心致歉**: LLM 道歉、叹气或用户倾诉不快时。双肩耷拉微驼背、视线躲闪低垂、双手交叉微摩搓。 |
| **8** | `agree_nod` | **Once** | ~60帧 (2s) | **认同点头**: 大模型说“我知道了 / 没问题”时瞬间触发。头部有节奏、快速有阻尼地连续点头 2 下，动作流落利索。 |
| **9** | `wave_disagree`| **Once** | ~60帧 (2s) | **否定摇头**: 模型表示“不是哦 / 不行”时触发。单手半抬手掌上下左右轻摇，头部快速摇头 2~3 下。 |

## ⚡ 三、 模型绑定与导出要求
1. **纯原地动画 (In-Place)**: 所有的动画角色都不产生水平方向真正的根坐标位移（保持 `Hips` 在 `0,0,0` 空间旋转起伏）。
2. **多余组件清理**: 剔除导出场景里多余相机、材质球、辅助刚体和高频静态骨骼冗余关键帧，单文件大小控制在 **5MB** 以内最佳。
3. **眼球独立骨骼**: 确保有 `LeftEye` / `RightEye` 眼神自驱动骨骼，以便在运行时用算法让角色实时对齐屏幕鼠标或摄像头。
